为什么需要它

直连多个 AI 平台的三大痛点

每个平台都有自己的 SDK、认证方式和接口格式，维护成本随平台数量指数增长。

🔌

接入复杂

DashScope、MiniMax、Moonshot 各自的 API 格式不一致，切换平台需要大量改动客户端代码，新增 Provider 成本极高。

💸

成本不可控

不知道每次请求实际落到哪个模型，简单问候误用昂贵推理模型，缺少统一的用量视图和预算控制。

🔇

上游异常难兜底

上游偶发的空回合（reasoning-only stop）、流式中断会直接透传给 Agent，导致「Agent couldn't generate a response」等难排查问题。

Bridge Server
→
统一接口 · 智能路由 · 异常兜底 · 成本可视
核心功能

不只是转发，更是智能适配层

从协议归一到 LLM 智能路由，Bridge Server 把零散的 AI 平台管理变成统一可控的工程能力。

🔀

五层智能路由

按优先级依次：客户端直接指定 → 旁路 LLM 路由器 → 代码插件路由器 → 场景正则路由 → ProviderManager 兜底，每层失败自动降级，路由决策全程可观测。

SmartRouter BypassRouter 插件路由器
🧠

旁路模型路由器

指定一个轻量 LLM 作为路由大脑，用自然语言描述路由规则，让 AI 决定每次请求应该交给哪个模型处理。超过阈值时还可自动压缩历史上下文。

LLM 路由决策 上下文压缩 超时降级
🛡️

空回合三层兜底

自动检测 reasoning-only stop、空 delta 等异常空回合，先尝试单次 non-stream 重试（附加强制回复提示词），仍失败则返回最小可见文本，不透传空响应给 Agent。

工具 follow-up 感知 自动重试 结构化日志
⚡

流式异常自动降级

流式请求在内容发送前失败时，自动降级为非流式调用并转换为 SSE 格式输出，客户端完全无感知。流式工具调用结构化解析，兼容 XML 格式工具调用。

stream→非流式 fallback tool_call 转换

实时监控 成本分析 热重载配置
🧪

模型能力基准测试

覆盖代码编程、数学推理、文学创作、语言翻译、日常对话五个维度，添加模型后一键发起测试，结果展示星级评分，帮助快速选出最适合业务的模型。

路由架构

五层优先级路由链

每一层独立决策，失败时自动传递给下一层，保证每个请求都能得到最合适的响应。

▸ 路由决策流程 — /v1/chat/completions
① 直接指定 model="dashscope/qwen3-coder-plus" — confidence 1.0，直达，跳过所有路由
↓ model 为空 / "smart"
② 旁路路由器 LLM 智能决策 + 上下文压缩 NEW
↓ 未启用 / 超时 / 决策失败
③ 插件路由器 用户 Python 插件，300ms 沙箱执行 NEW
↓ 无激活插件 / 执行失败
④ 场景路由 正则匹配 coding/writing/translation 等场景，支持优先级和排除规则
↓ 无匹配场景
⑤ ProviderManager 按 fallback / round_robin / cost_optimized / lowest_latency 策略兜底

快速开始

5 分钟完成部署

选择最适合你的安装方式，安装后访问 http://localhost:19377 进入管理面板。

pip 直接运行
Setup Wizard
Docker
bash
# 1. 安装
pip install bridge-server
 
# 2. 设置 API Key（至少一个 Provider）
export DASHSCOPE_API_KEY=sk-xxx
 
# 3. 启动
python -m uvicorn bridge_server.runtime:app --app-dir src --port 19377
 
# 4. 发起第一个请求
curl -X POST http://127.0.0.1:19377/v1/chat/completions \
-H "Authorization: Bearer <your-token>" \
-H "Content-Type: application/json" \
-d '{"model":"smart","messages":[{"role":"user","content":"你好"}]}'
💡 首次启动会自动生成 Admin Token 并打印到控制台，同时自动打开管理面板（http://localhost:19377）。
